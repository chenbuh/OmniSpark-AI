# 接口防爬取与防滥用加固清单

更新时间：2026-06-07

## 当前已落地

1. 登录 / 注册前置自研验证码，必须先拿一次性 `captchaTicket` 才能进认证接口。
2. 密码传输已改为前端公钥加密、后端解密，默认不再接受明文密码。
3. 敏感写请求已接入服务端 challenge 签名，降低脚本重放价值。
4. 登录接口已有 IP 维度限流。
5. 登录失败已有短时锁定，按 `用户名`、`用户名 + IP`、`设备 + 用户名` 三个维度累计失败。
6. 管理员登录已改成两段式：
   第一段先校验账号密码。
   第二段必须过 TOTP。
   首次管理员登录必须先绑定 TOTP 才发 token。

## 按收益排序的加固清单

1. 管理员账号强制 TOTP，不允许关闭。
   收益最高，直接拦住“撞到密码就进后台”的风险。
   当前状态：已完成。

2. 登录失败锁定继续收紧。
   建议阈值：同用户名 8 次 / 15 分钟，同用户名 + IP 5 次 / 10 分钟，同设备 + 用户名 6 次 / 20 分钟。
   当前状态：已完成，可按线上误伤情况继续调阈值。

3. 登录、注册、验证码接口继续做分层限流。
   匿名入口至少保住 `IP` 维度，已登录高风险接口再叠 `USER_API`。
   当前状态：登录 / 注册 / 验证码已接；后续建议把更多公共读接口补齐。

4. 管理后台路径单独限流。
   `/api/admin/**` 不要沿用普通用户接口阈值，建议更严。
   当前状态：部分高风险接口已有签名，限流还可以继续加密。

5. 公开列表接口分页上限统一夹紧到 100 以内。
   这是最便宜的防批量拖库手段，能直接压低单次抓取效率。
   当前状态：需要逐个核对未夹紧的控制器。

6. 社区 / 模板 / 风格卡接口做字段最小化。
   列表只给卡片必需字段，详情再给完整内容。
   当前状态：未系统收口。

7. 生成接口增加“用户在跑任务数”上限。
   限流能挡瞬时洪峰，在跑任务上限能防单人占满线程池。
   当前状态：频率限流已做；并发上限建议补 Redis 计数。

8. 登录态与 API Key 风险联动。
   同一账号短时跨多个 IP / UA 异常切换时，直接抬高风控等级或要求二次验证码。
   当前状态：已有访问日志，可继续联动策略。

9. 上传与静态资源加防盗链和下载限频。
   重点保护 `uploads/**`、导出接口、社区图片。
   当前状态：未完成。

10. 生产环境强制 HTTPS + HSTS。
    没有这层，前面的传输保护和签名价值都会被削弱。
    当前状态：需要运维侧落地。

11. 补安全响应头。
    至少加 `X-Frame-Options`、`X-Content-Type-Options`、`Content-Security-Policy`、`Referrer-Policy`。
    当前状态：未完成。

12. 做异常访问告警。
    重点盯登录失败暴涨、验证码失败暴涨、单 IP 高频读接口、单用户高频生成。
    当前状态：日志基础有了，告警规则还没配。

## 建议的下一批优先级

1. `/api/admin/**` 单独限流。
2. 全量检查分页上限。
3. 公开接口字段最小化。
4. 生成任务并发上限。
5. 安全响应头 + HTTPS 落地。

## 这版改造的核心结论

1. 普通脚本现在已经不能再靠“账号密码 + 批量请求”直接自动化登录后台。
2. 现在最有价值的后续动作，不是继续堆花哨混淆，而是把后台限流、分页上限、公开数据最小化补完整。
