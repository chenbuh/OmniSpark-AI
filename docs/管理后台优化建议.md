## 2. 🔴 严重问题

### 2.1 ✅ 审计日志接口缺管理员校验(水平越权)

**问题**
`AuditController.java:17` 类级只有 `@SaCheckLogin`,没有 `@SaCheckRole("admin")`。`list()`(`:20-27`)接受任意 `userId`/`action` 参数且无归属校验。任何登录的普通用户都能调用 `GET /api/audit-logs?userId=<任意ID>` 读取他人的全部审计记录。`/my`(`:29-36`)用当前登录 ID 是正确的,但 `list` 暴露了全量。

**影响**:全站审计数据(谁在何时做了什么、IP 等)对任意登录用户横向泄露。

**位置**:`module/audit/AuditController.java:17`、`:20-27`

**修复方向**:`list()` 方法或类上补 `@SaCheckRole("admin")`,`/my` 保持仅查本人。

**✅ 修复状态(2026-05-30):已修复**
- `AuditController.list()` 加 `@SaCheckRole("admin")`,全量查询仅管理员可用。
- 前端 `AuditLog.vue`(普通用户菜单)改调 `/api/audit-logs/my` 仅看本人记录,并归并到统一 `request` 封装(去掉手写 `fetch` 与硬编码 `localhost:8080`)。

### 2.2 ✅ 数据清理无下限校验,可一键清空全库

**问题**
`AdminCleanupController.java:56-78` 的 `execute(daysOld)` 用 `LocalDateTime.now().minusDays(daysOld)` 计算 cutoff,`daysOld` 无 `>0` 校验:
- 传 `daysOld=0` → cutoff = 现在,删除几乎所有历史数据;
- 传**负数**(如 `-3650`) → `minusDays(-3650)` 得到未来时间,`lt(future)` 命中**所有行**。

清理横跨 `generation_task` / `asset` / `audit_log` / `login_log` 四表做物理 `delete`,且整个方法**无 `@Transactional`**,部分失败会留下不一致状态。

**影响**:一次误操作或被诱导的请求即可造成不可逆的全量数据丢失。

**位置**:`module/system/AdminCleanupController.java:55-78`

**修复方向**:
1. 强制 `daysOld` 下限(如 `>= 7`),非法值直接拒绝;
2. 方法加 `@Transactional(rollbackFor = Exception.class)`;
3. (增强)要求前端传二次确认参数,避免误触。

**✅ 修复状态(2026-05-30):已修复(1、2)**
- 新增 `MIN_DAYS_OLD=7` 与 `requireValidDaysOld()`,`preview`/`execute` 均校验,`daysOld < 7`(含 0、负数)直接抛业务异常拒绝。
- `execute` 加 `@Transactional(rollbackFor = Exception.class)`,四表删除要么全成功要么回滚。
- 第 3 项(二次确认参数)为可选增强,暂未实施。

### 2.3 ✅ 日志查看器路径穿越(管理员上下文内任意文件读取)

**问题**
`LogViewerController.java:36` 用 `Paths.get(logPath, file)` 直接拼接用户传入的 `file` 参数,**未做 normalize / 目录包含校验**(对比 `FileManagerController` 的上传场景做了 `startsWith` 校验,这里完全没有)。传 `file=../../../../etc/passwd` 或 Windows 绝对路径即可逃逸 `logs` 目录,`Files.exists` 通过后逐行读出返回。

类级已有 `@SaCheckRole("admin")`,因此利用前提是已持有管理员令牌——风险定级为「管理员账户被盗/越权后可读服务器任意文件」,仍属严重纵深问题。

**影响**:配合管理员令牌泄露,可读取服务器上任意可读文件(配置、密钥、系统文件)。

**位置**:`module/system/LogViewerController.java:36`

**修复方向**:
```java
Path base = Paths.get(logPath).toAbsolutePath().normalize();
Path logFile = base.resolve(file).normalize();
if (!logFile.startsWith(base) || !file.endsWith(".log")) {
    throw new BusinessException("非法的日志文件名");
}
```

**✅ 修复状态(2026-05-30):已修复**
- `LogViewerController` 改为 `base.resolve(file).normalize()` 并校验 `startsWith(base)` + `.log` 后缀,非法文件名直接返回「非法的日志文件名」,杜绝穿越读取任意文件。

## 6. 功能完善方向(非缺陷,增强建议)

- **操作审计闭环**:管理后台的删除/清理/角色变更等高危操作,建议统一经 `AuditAspect` 落审计日志,便于事后追溯。
- **仪表盘下钻**:`AdminDashboard` 的统计卡片可支持点击下钻到对应明细列表(如点「失败任务数」跳任务列表并预置 status 过滤)。
- **批量操作**:用户/任务/资产列表补多选 + 批量删除/禁用,减少逐条操作。
- **配置热加载**:`SystemConfig` 修改后若能即时生效(而非重启),可提升运维体验。
- **监控告警**:`MonitorController` 已有 CPU/内存/磁盘数据,可加阈值告警(如磁盘 >90% 推通知),而非仅展示。
