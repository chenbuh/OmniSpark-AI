# 假接口与假数据清理清单

更新时间：2026-06-07

## 目标

本清单用于梳理当前仓库中仍然存在的“假接口 / 假数据 / 伪参数 / 占位实现”，并区分：

- 必须清理的未完成实现
- 可以保留但需要改名或改文案的静态配置接口
- 不应误判为假数据的系统初始化数据

本次结论基于当前代码状态人工核查得出，不依赖历史记忆。

## 排查范围

- 前端：`src/api`、`src/store`、`src/views`、`src/components`
- 后端：`backend/src/main/java`
- 数据结构与初始化：`backend/src/main/resources/schema.sql`、`backend/src/main/java/.../bootstrap`

## 总结

- 当前项目**没有发现大面积前端 mock 列表接口**。
- 真实需要清理的重点集中在：
  - 项目导入导出的占位实现
  - 提示词模板 / 风格卡保留了“项目维度外观”，但底层已经改成公共库
  - 管理端任务筛选元数据曾由历史任务反推，表现得像配置接口但实际受存量数据偶然性影响
  - 审计日志操作类型筛选曾由历史日志反推，表面像固定动作目录，实际受已有记录影响
- 另外存在一批 `meta` 类接口，它们**不是假接口**，但返回的是**静态枚举配置**，后续如果要做成运营可配，需要迁移到配置中心或数据字典。
